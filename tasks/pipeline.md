# Конкурентный пайплайн
Необходимо реализовать функцию для запуска конкурентного пайплайна, состоящего из стейджей.

```go
package pipeline

import (
	"context"
)

type (
	In  = <-chan any
	Out = In
)

type Stage func(ctx context.Context, data any) error

type Pipeline interface {
	ExecutePipeline(ctx context.Context, in In, stages ...Stage) Out
}

```

## Требования
Обработка последующего элемента входных данных должна происходить без ожидания завершения всего пайплайна для текущего элемента.

Т.е. пайплайн из 4 функций по 100 мс каждая для 5 входных элементов должен выполняться гораздо быстрее, чем за 2 секунды (4 * 100 мс * 5).

- Должно быть реализовано корректное логирование работы всех этапов
- Ошибка при обработке не должна приводить к потере остальных данных

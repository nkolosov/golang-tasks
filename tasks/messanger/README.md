# Мессенджер

Необходимо спроектировать и реализовать простой групповой мессенджер.

Клиент инициализирует собственную сессию, получая уникальный UUID, после чего может создать или присоединиться к уже существующему чату, видеть сообщения в нем, отправлять свои.

Проект должен соответствовать [базовым требованиям](../../README.md)

## Функциональные требования
- При создании чата можно указать время его жизни
- При создании чата можно указать максимальное количество сообщений в нём (в логике LRU)
- Можно создать read-only чат (тогда писать может только создатель чата)
- При запуске сервера можно указать максимальный размер одного чата и максимальное количество чатов и сессий (через флаги)
- Хранение данных in-memory, с возможностью в дальнейшем использовать другой тип хранилища

## Требования к реализации
- Проект должен запускаться в Docker
- Чат должен поддерживать gRPC и HTTP API
- Разработка должна вестись по принципу spec-first (сервер и клиенты должны генерироваться через oapi-codegen)
- Должен быть реализован корректный Graceful Shutdown
- Должно быть настроено корректное логирование и обработка ошибок, сбор базовых Prometheus-метрик.
